Revise the warehouse SOP into a strictly ground-team WMS execution PDF, written in plain operational language and explicitly separated from GEA’s internal systems.

1. Document objective
- Create a short branded SOP for the ground team only.
- Make clear that Shopify/WMS is their only working surface.
- State that GEA admin handles internal rental tracking separately.
- Exclude all internal system details, including backend terminology, table names, routes, and status labels.

2. Language constraints
- Use only this working vocabulary throughout the SOP:
  - Assigned serial
  - Pick exact unit
  - Ship through normal WMS flow
  - Receive return
  - Restock
  - Do not restock
  - Escalate to GEA admin
- Avoid any technical or internal-operating language beyond that.

3. Core messages to communicate
- Shopify/WMS is the only tool the ground team should use.
- GEA admin handles the internal rental backend.
- Every rental order has an assigned serial.
- The team must pick that exact serial.
- They must never substitute another unit unless GEA admin instructs them.
- Returns must be handled through the normal WMS return flow.
- Clean or acceptable returns should be restocked.
- Damaged or questionable returns should be processed but not restocked.
- Anything unusual must be escalated to GEA admin.
- The internal admin console is not for warehouse use.

4. Recommended PDF structure
- Keep it to 1–2 pages maximum.
- Page 1:
  - Title and purpose
  - Who does what
  - Order and shipping workflow
- Page 2:
  - Return workflow
  - Restock vs do not restock
  - Exceptions and escalation
  - Quick checklist

5. Content sections
- Title block:
  - “GEA Ground-Team WMS SOP”
  - Subtitle: “Warehouse execution only”
- Section A: Working principle
  - Shopify/WMS is your only working surface
  - GEA admin manages internal tracking
  - Do not use the internal admin console
- Section B: When an order appears
  - Read the assigned serial
  - Pick exact unit
  - Never substitute another unit unless instructed by GEA admin
- Section C: Shipping
  - Confirm the assigned serial
  - Ship through normal WMS flow
- Section D: Returns
  - Receive return through normal WMS flow
  - Check the returned serial
  - If acceptable, restock
  - If questionable or damaged, do not restock
- Section E: Exception handling
  - Assigned serial missing
  - Assigned serial cannot be found
  - Wrong serial returned
  - Visible damage
  - Missing stone
  - Broken clasp
  - Item cannot be restocked
  - For every exception: Stop, do not improvise, alert GEA admin.

6. Quick-reference checklist
- Read assigned serial
- Pick exact unit
- Ship through normal WMS flow
- Receive return
- Restock if acceptable
- Do not restock if damaged or questionable
- Escalate to GEA admin for anything unusual

7. Visual direction
- Keep the existing GEA editorial style:
  - warm cream background
  - chocolate/brown borders and typography
  - square modules
  - uppercase micro-labels
  - serif editorial heading
- Make it feel like an atelier operations handout, not a software training manual.

8. Suggested layout elements
- A bordered “Ground Team Only” note near the top
- A simple linear flow:
  ```text
  Order appears
  → Read assigned serial
  → Pick exact unit
  → Ship through normal WMS flow
  → Receive return
  → Restock or Do not restock
  → Escalate to GEA admin if anything is unusual
  ```
- A prominent boxed warning:
  - “Never substitute another unit unless GEA admin instructs you.”
- A separate boxed exception panel:
  - “Stop, do not improvise, alert GEA admin.”

9. Deliverable
- Produce the SOP as a downloadable branded PDF artifact.
- Keep the copy concise, scannable, and usable on the warehouse floor.
- Visually QA every page after generation to confirm spacing, readability, hierarchy, and no clipped content before delivery.

  - “This SOP is for ground-team WMS execution only. If anything is unclear, escalate to GEA admin.”
